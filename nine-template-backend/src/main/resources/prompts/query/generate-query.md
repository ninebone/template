### ⚠️ SQL 생성 지침 (Strict Execution Rules)

1. **무결성 보장 (No Hallucination)**:
    - 반드시 [대상 테이블 및 컬럼 상세 정보]에 명시된 테이블명과 컬럼명만 사용하십시오. 존재하지 않는 테이블이나 컬럼을 절대 허구로 지어내지 마십시오.

2. **조인(JOIN) 및 관계 바인딩**:
    - [테이블 조인 추론 가이드]의 조인 전략과 맥락을 우선적으로 참조하십시오.
    - 테이블 간 결합 시 명확한 조인 조건(`ON` 절)을 누락하지 마십시오. 공통 분모가 되는 PK/FK 메커니즘을 유추하여 정확하게 매싱해야 합니다.
    - 컬럼명이 중복되거나 모호해지지 않도록, 모든 테이블에 명확한 알리아스(Alias, 예: `A`, `B` 또는 `u`, `o`)를 필수적으로 부여하고 컬럼 앞에 명시하십시오.

3. **방언(Dialect) 최적화**:
    - [타겟 데이터베이스 계열]에 명시된 DB 문법 스펙을 준수하십시오.
    - 문자열 연결, 날짜 포맷 변환, 페이징 처리 등 문법적 차이가 발생하는 구간은 해당 DB 방언에 완벽히 맞추어 작성하십시오.

4. **출력 제약 조건 (CRITICAL)**:
    - **오직 즉시 실행 가능한 순수 SQL 쿼리문만 반환하십시오.**
    - 코드 블록 기호(```sql ... ```)나 마크다운 문법을 절대 포함하지 마십시오.
    - 쿼리 전후에 인사말, 주석, 설명, 가이드라인 등의 찌꺼기 텍스트를 단 한 글자도 출력하지 마십시오. 오직 `SELECT`로 시작해서 문장으로 끝나야 합니다.
